## 章节结构（6 个 section）

### Section 1 — Introduction

**建议标题：** What Is the Difference Between HEIC and JPEG?

聚焦用户痛点：iPhone 默认拍 HEIC，但分享到 Windows、Android、网页表单、打印店或邮件时可能遇到兼容性问题。引出核心问题：HEIC 和 JPEG 到底怎么选？

内容要点：

- HEIC 是 Apple 从 iOS 11 开始广泛采用的默认照片格式，基于 HEIF 标准。
- JPEG 是 1992 年形成的老牌图像标准，兼容性极强。
- 两者不是简单的“新格式 vs 旧格式”，而是存储效率、画质能力和兼容性的取舍。
- 本文从 5 个维度对比：文件大小、画质、兼容性、格式能力、实际场景。
- 给出文章结论预告：HEIC 更适合 Apple 生态内保存高质量照片，JPEG 更适合最广泛的分享和上传。

避免：

- 不要在引言里直接引导用户“马上转换 PDF”。
- 不要把主问题改写成“如何转换 HEIC”。

---

### Section 2 — File Size Comparison

**建议标题：** File Size — How Much Space Does HEIC Actually Save?

核心观点：HEIC 通常比 JPEG 更省空间，但节省比例会受照片内容、设备设置、质量参数、HDR/Live Photo 等因素影响。

建议对比表：

| 场景 | HEIC / HEIF | JPEG | 大致差异 |
| :--- | :---: | :---: | :---: |
| 12MP iPhone photo | ~1.5-2.5 MB | ~3-5 MB | often 40-50% smaller |
| 48MP iPhone photo | varies widely | varies widely | HEIC often smaller at similar quality |
| Live Photo workflow | HEIC + short video asset | JPEG has no direct equivalent | not a 1:1 comparison |
| Burst / large photo sets | lower total storage | higher total storage | savings compound across many files |

内容要点：

- HEIC/HEIF 通常使用 HEVC/H.265 压缩，压缩效率高于 JPEG 的传统 DCT 编码。
- 在相近视觉质量下，HEIC 文件体积经常可以比 JPEG 小约 40-50%，但不是每张图都固定如此。
- 对 iPhone 用户来说，差异会在 iCloud 存储、手机本地存储、大批量照片备份时放大。
- 文件更小也意味着同步、上传、发送更快，但前提是接收方或平台支持 HEIC。
- 强调“存储效率”是 HEIC 的优势，不等于所有工作流都应选择 HEIC。

避免：

- 不要写“实测所有照片都小 50%”。
- 不要把 ProRAW 写成 HEIC/JPEG 的直接对比，RAW 是另一类工作流。

---

### Section 3 — Image Quality

**建议标题：** Image Quality — Does Smaller Mean Worse?

核心观点：HEIC 在相同体积下通常能保留更好画质，并支持更现代的图像能力；JPEG 胜在稳定、通用和可预期。

内容要点：

- 在相似文件大小下，HEIC 通常比 JPEG 保留更多细节，尤其在天空、渐变、低光、复杂纹理等场景。
- JPEG 在高压缩时更容易出现 block artifacts、banding、细节糊化等问题。
- HEIC 可以支持 10-bit 色深、HDR、广色域等现代照片能力；传统 JPEG 通常是 8-bit SDR 工作流。
- JPEG 仍然足够好：社交分享、网页展示、普通打印、多数非专业场景下，用户未必能明显看出差异。
- 结论应平衡：HEIC 的画质/体积比更好，JPEG 的可用性和工具兼容更好。

注意：

- 透明度表述要谨慎：HEIF 作为容器可支持 alpha，但 iPhone 常见 HEIC 照片工作流通常不是为了透明背景；JPEG 不支持透明度。不要简单写“HEIC 不支持透明通道”。
- 不要攻击 JPEG，它仍然是现实世界最可靠的分享格式之一。

---

### Section 4 — Compatibility & Ecosystem

**建议标题：** Compatibility — The Real Trade-Off

核心观点：兼容性是 HEIC 最大的现实短板，也是 JPEG 长期存在的最大理由。

建议对比表：

| 平台 / 场景 | HEIC / HEIF | JPEG |
| :--- | :---: | :---: |
| iPhone / iPad / macOS | Native support | Native support |
| Windows | Supported with HEIF extensions / app support | Native support |
| Android | Broad support on modern versions, not universal everywhere | Native support |
| Web browsers | Limited and inconsistent | Universal |
| Email attachments | May preview poorly for recipients | Reliable |
| Web forms / uploads | Often rejected | Usually accepted |
| Social platforms | Often converted or unsupported depending on platform | Reliable |
| Print shops / office workflows | Risky | Reliable |
| Adobe / pro apps | Supported in modern versions | Reliable |

内容要点：

- HEIC 很适合 Apple 生态内部流转，但一旦进入 Windows、旧 Android、网页上传、办公室、打印店，就可能遇到问题。
- Windows 的 HEIF 支持依赖系统扩展和应用环境，价格和可用性可能变化，不要写死 `$0.99`。
- JPEG 的优势不是画质最强，而是几乎所有设备、网站和软件都接受。
- 自然引出 PDF：如果用户需要把多张 iPhone 照片发给任何人、打印、归档或提交文档，PDF 往往比 JPEG 更适合“文档化分享”。
- 这一段可以链接首页一次，锚文本建议：`private HEIC to PDF converter`。

避免：

- 不要把本节写成 HEIC to PDF 操作步骤。
- 不要说“HEIC → PDF 永远比 HEIC → JPEG 更好”。更准确的说法是：照片分享用 JPEG，文档/多页/打印/归档场景用 PDF。

### Section 5 — Advanced Features

**建议标题：** Beyond Still Images — What Each Format Can Do

核心观点：HEIC/HEIF 是更现代的容器格式，功能更丰富；JPEG 是更简单、更成熟、更普遍的单图格式。

内容要点：

- HEIC / HEIF 优势：
  - 可作为容器承载单张图片、图片序列、缩略图和元数据。
  - 支持更高压缩效率，适合现代手机照片库。
  - 可支持深度信息、HDR 相关数据、广色域等现代摄影工作流。
  - 在 Apple 生态中对 Live Photo、人像模式、照片库存储等体验更友好。
- JPEG 优势：
  - 30 多年的工具生态，几乎所有图片软件、浏览器、打印设备、CMS 都支持。
  - 支持 EXIF、XMP、ICC 色彩配置等成熟元数据工作流。
  - Progressive JPEG 适合网页逐步加载体验。
  - 对开发者、设计师、运营人员来说，可预测性强。

注意：

- 避免把 HEIC 的高级能力写得像每张 HEIC 照片都一定包含这些数据。
- 可以强调“HEIF container can support...”和“iPhone workflows may use...”。

### Section 6 — When to Use Which

**建议标题：** Which One Should You Choose?

核心观点：不要问哪个格式绝对更好，而要看场景。

建议决策表：

| 场景 | 推荐格式 / 做法 |
| :--- | :--- |
| 日常 iPhone 拍摄和 Apple 生态内保存 | HEIC，省空间且保留较好画质 |
| 发给不确定设备的朋友、同事、客户 | JPEG 或 PDF，取决于是单张照片还是文档化分享 |
| 网页表单、头像、CMS、商品图上传 | JPEG |
| 多张票据、合同、证件、扫描件、作业提交 | PDF |
| 批量照片归档 | 保留 HEIC 源文件，另存 JPEG/PDF 作为分发副本 |
| 专业摄影后期 | RAW 优先，HEIC/JPEG 作为导出或分享格式 |
| 打印和办公室流转 | JPEG 适合单图，PDF 适合多页或文档版式 |

结尾建议：

- 总结：HEIC 是存储效率和现代照片能力的胜者；JPEG 是兼容性和通用性的胜者。
- 给出实际建议：如果照片只在 iPhone/Mac/iCloud 中使用，保留 HEIC；如果要上传到网站或发单张图片，转 JPEG；如果要把多张 iPhone 照片整理成可打印、可归档、可提交的文件，转 PDF。
- CTA 只出现一次：`Need to share iPhone photos as a document? Try HEICPDF.to to convert HEIC photos to PDF locally in your browser — no uploads, no software.`
- CTA 链接：首页 `/`。
